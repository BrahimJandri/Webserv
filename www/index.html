About WebServ

WebServ is a custom-built HTTP server written in C++98 designed to process web requests efficiently while maintaining full non-blocking I/O operations. It serves static files, executes CGI scripts for dynamic content, manages sessions and cookies, and supports file uploads and deletions.

Key design goals include robustness, concurrency, and compliance with HTTP standards, making WebServ a practical and educational project that explores low-level network programming, event-driven I/O, and server architecture.

Developed with Unix system calls like poll(), socket(), fork(), and others, WebServ emphasizes non-blocking behaviour and proper resource management to handle multiple simultaneous clients without blocking or unnecessary CPU usage.

Core Features & HTTP Compliance

Multi-port Listening:

Serve multiple websites or services by listening on various interface:port combinations.

Non-blocking I/O:

All network communication managed using poll() without blocking.

HTTP Methods:

Supports GET, POST, and DELETE with proper status codes.

CGI Execution:

Runs external scripts (PHP, Python, Bash) with full environment management.

Although full HTTP RFC compliance is not mandatory, WebServ aligns closely with HTTP/1.0 and HTTP/1.1 standards, making it compatible with common browsers and tools like NGINX for behavior comparison.
